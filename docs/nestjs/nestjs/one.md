## 📘 第 0 课：先把“认知”对齐（非常重要）

在开始写代码前，你要记住一句话：

> **NestJS = 架构思想 + TypeScript + Express/Fastify**

### 和 Express 最大的区别

| Express  | NestJS   |
| -------- | -------- |
| 面向过程     | 面向架构     |
| 自由但混乱    | 约束但清晰    |
| 文件随意     | 模块强约束    |
| 中大型项目难维护 | 天然适合大型项目 |

👉 **NestJS 不是为了快，是为了“长期可维护”**

---

## 🧠 NestJS 的核心抽象（你现在先记住这 5 个）

```txt
Request
  ↓
Controller   （接收请求）
  ↓
Service      （业务逻辑）
  ↓
Repository   （数据库）
```

再加 2 个：

* **Module**：组织代码的“容器”
* **Provider**：一切可被注入的东西
