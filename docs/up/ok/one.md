# 二、进入端口转发（关键步骤）

## 1️⃣ 点击右侧按钮：**端口转发(P)**

会弹出一个新窗口，通常叫：

```
端口转发规则
```

里面现在应该是 **空的**（没有任何规则）。

---

## 2️⃣ 新增一条 SSH 转发规则（照抄）

在端口转发窗口里：

* 点击右侧 **➕（添加新规则）**

然后填写下面这一行（非常重要，逐项照填）👇

| 字段     | 填什么      |
| ------ | -------- |
| 名称     | ssh      |
| 协议     | TCP      |
| 主机 IP  | **留空**   |
| 主机端口   | **2222** |
| 子系统 IP | **留空**   |
| 子系统端口  | **22**   |

> ⚠️ 注意
>
> * **IP 两个都留空**（这是 VirtualBox 官方推荐写法）
> * 端口号不要写反

填完后你会看到一行类似：

```
ssh | TCP |  | 2222 |  | 22
```

---

## 3️⃣ 保存设置

* 点击 **确定**
* 再点击一次 **确定**
* 关闭设置窗口

---

# 三、重新启动虚拟机（必须）

端口转发 **只有在虚拟机重新启动后才生效**。

### 正确做法：

1. 如果虚拟机在运行 → **关闭电源**
2. 回到 VirtualBox 主界面
3. 再次启动 `ub-2204`

---

# 四、从宿主机 SSH 连接（现在做）

在你 **自己的电脑（不是虚拟机）**上：

### Windows（PowerShell） / macOS / Linux 终端：

```bash
ssh -p 2222 wjx@127.0.0.1
```

或者：

```bash
ssh -p 2222 wjx@localhost
```

### 第一次连接时：

```
Are you sure you want to continue connecting (yes/no)?
```

输入：

```
yes
```

然后输入密码。

# 五、如果 SSH 失败（不用慌）

常见原因只有 3 个：

1. 忘了重启虚拟机
2. 端口号写错（2222 ↔ 22）
3. 主机端口被占用（极少）

如果失败，把**完整报错内容**原样发给我，我直接帮你定位。
